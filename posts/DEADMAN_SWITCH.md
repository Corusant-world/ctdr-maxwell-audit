# DEADMAN SWITCH (Public‑safe teaser) — если NVIDIA молчит

**Цель:** создать давление “рынок движется без вас”, но без токсичности, без угроз, без спама, без утечки IP.  
**Формула:** один сильный публичный ход + один сильный инженерный канал (GitHub/Forums) + чистый NDA‑путь.

---

## Триггер

Запускаем deadman‑эскалацию если выполнены оба условия:

1) **Тишина 72 часа** после первого сообщения по инженерному треку (нет ни одного “routing reply”).  
2) Нет назначенного **20–30 мин** technical screen.

**Отсчёт в этой кампании:** старт **2025‑12‑18 21:00** → deadman наступает **2025‑12‑21 21:00**.

---

## Ограничения (обязательные)

- **Никаких** DPX kernel source / PTX / SASS / bindings.
- **Никакого** репо‑хэндовера до definitive agreement.
- Публично: только **public‑safe**: метрики, логи, графики, высокоуровневая механика.

---

## Deadman‑ходы (порядок исполнения)

### 1) One decisive forcing move: Public priority + evaluation window (один раз)
Это не “ещё один подход”. Это объявление рынку:

- **Приоритет зафиксирован** (arXiv / preprint package).
- **Окно оценки ограничено по времени** (до Dec 31): мы ищем 1 owner + NDA‑оценку сейчас.
- **Scarcity / moat для NVIDIA**: узкий field‑of‑use эксклюзив (use‑it‑or‑lose‑it).

Цель: заставить внутреннюю маршрутизацию NVIDIA сработать (появляется “owner or I miss the window”).

Документ‑скелет тизера/вызова: `posts/INFINITE_CONTEXT_CHALLENGE.md`.
Forcing package runbook: `posts/FORCING_PACKAGE_RUNBOOK.md`.

### 2) GitHub routing (1–2 поста, не больше)
- Открываем **1 issue** в `NVIDIA/TensorRT-LLM` и/или **1 discussion/issue** в `triton-inference-server/server`.
- Тон: routing ask → назначить owner → 20–30 min screen → NDA → контейнер/runbook.
 

### 3) NVIDIA Developer Forums routing (hard cap: 2 total)
- Ровно 2 реплая в официальных тредах (без новых тем), routing ask only.
 

### 4) One‑time CEO routing (только если 0 ответов)
- Одно письмо ≤10 строк, без вложений, без “AGI”.
- Только routing к инженерному owner.
